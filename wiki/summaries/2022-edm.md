# 拡散ベース生成モデルの設計空間の解明（EDM）

> 原典: [[translations/2022-edm]] ・ `raw/papers/Elucidating the Design Space of Diffusion-Based Generative Models.pdf`
> 著者・年・会議: Karras, Aittala, Aila, Laine（NVIDIA）・2022・NeurIPS 2022（arXiv:2206.00364。通称 **EDM**）

## 一言まとめ

拡散モデルの設計を「**サンプリング・ネットワーク前処理（preconditioning）・学習**」という**独立に最適化できるモジュール空間**に分解し、各成分のベストプラクティス（**Heun 2 次サンプラー＋ρ=7 の σ スケジュール＋churn 確率サンプラー**、第一原理から導いた **preconditioning** $c_{\rm skip}/c_{\rm out}/c_{\rm in}/c_{\rm noise}$、**対数正規ノイズ分布** $P_{\rm mean}=-1.2,P_{\rm std}=1.2$）を確立した「設計空間の整理＋実用改善」論文。CIFAR-10 で FID 1.79（cond）/1.97（uncond）、ImageNet-64 で 1.36 を、わずか 35 NFE（ネットワーク評価回数）で達成。後続の SD3・SDXL・Stochastic Interpolants が土台として参照する。

## 背景と問題意識

拡散モデルの文献は理論的に密で、ノイズスケジュール・学習ダイナミクス・パラメータ化を可能な限り理論的基盤から直接導く傾向があった。これは堅固だが、**個々の成分を全体を壊さずに変更できない密結合パッケージ**にモデルを見せてしまう危険がある。例えば VP（DDPM 系）・VE（NCSN 系）・iDDPM は別々の理論的枠組みから来ており、設計選択が絡み合って比較・改善が難しかった。

EDM は「理論的過程」より「学習・サンプリングに現れる具体的な対象・アルゴリズム」に焦点を当て、これらを **1 つの共通枠組み（表1）に並べる**ことで、どの成分が独立に動かせるかを露わにした。前提は [[score-based-generative-models]] の denoising score matching：denoiser $D(\boldsymbol{x};\sigma)$（ノイズレベル $\sigma$ でノイズ除去）が学習でき、スコアは $\nabla_{\boldsymbol{x}}\log p(\boldsymbol{x};\sigma)=(D(\boldsymbol{x};\sigma)-\boldsymbol{x})/\sigma^{2}$ で得られる。

## 提案手法 / 主張

### 1. 共通枠組み（§2, Table 1）

拡散の生成を確率フロー ODE $\mathrm{d}\boldsymbol{x}=-\dot{\sigma}(t)\sigma(t)\nabla_{\boldsymbol{x}}\log p(\boldsymbol{x};\sigma(t))\mathrm{d}t$ と、スケジュール $\sigma(t)$・スケーリング $s(t)$・時間離散化 $\{t_i\}$・preconditioning・学習設定の組として表現。**VP/VE/iDDPM+DDIM/EDM をすべて同じ表の列として整理**（[[probability-flow-ode]] の Score-SDE をさらに「設計選択の分解」へ進めた）。推奨は **$\sigma(t)=t$, $s(t)=1$**——これで σ と t が交換可能になり、ODE が $\mathrm{d}\boldsymbol{x}/\mathrm{d}t=(\boldsymbol{x}-D(\boldsymbol{x};t))/t$ に簡約、解軌道がほぼ直線化して少ステップ化に効く。

### 2. 決定論的サンプリング（§3, Algorithm 1）

- **Heun 2 次法**（台形補正）：Euler（1 次, 局所誤差 $\mathcal{O}(h^2)$）より、1 回追加の $D_\theta$ 評価で $\mathcal{O}(h^3)$。打ち切り誤差/NFE のトレードオフが優秀。
- **ρ スケジュール**：$\sigma_{i}=(\sigma_{\rm max}^{1/\rho}+\frac{i}{N-1}(\sigma_{\rm min}^{1/\rho}-\sigma_{\rm max}^{1/\rho}))^{\rho}$、**ρ=7**（低ノイズ側のステップを短く）。
- 事前学習済み VP/VE/DDIM にサンプラーを差し替えるだけで NFE を **7.3×/300×/3.2×** 削減。これがサンプリングと学習が直交するという主張の実証。

### 3. 確率サンプリング（§4, Algorithm 2）

ODE に Langevin 的「**churn（攪拌）**」を交互適用：各ステップでノイズを少し足して σ を一時的に上げ（$\hat{t}_i=t_i+\gamma_i t_i$）、ODE で下げる。これは先のステップの誤差を**自己訂正**する。$S_{\rm churn}/S_{\rm tmin}/S_{\rm tmax}/S_{\rm noise}$ で制御。過剰だと detail loss・過飽和（$L_2$ 学習 denoiser が非保存的場を誘導するため）。サンプラーのみで ImageNet-64 を FID 2.07→1.55。

### 4. Preconditioning と学習（§5, Table 2）

ネットを直接 $D_\theta$ にせず、$\sigma$ 依存の skip 接続で前処理：

$$
D_\theta(\boldsymbol{x};\sigma)=c_{\rm skip}(\sigma)\boldsymbol{x}+c_{\rm out}(\sigma)F_\theta(c_{\rm in}(\sigma)\boldsymbol{x};c_{\rm noise}(\sigma)).
$$

入力と学習目標を**単位分散**に保ち、ネット $F_\theta$ の誤差増幅を最小化するよう $c_{\rm in}=1/\sqrt{\sigma^2+\sigma_{\rm data}^2}$, $c_{\rm skip}=\sigma_{\rm data}^2/(\sigma^2+\sigma_{\rm data}^2)$, $c_{\rm out}=\sigma\sigma_{\rm data}/\sqrt{\sigma^2+\sigma_{\rm data}^2}$ を**第一原理で導出**（[[diffusion-model-architecture]] のネット入出力設計軸）。損失重み $\lambda(\sigma)=1/c_{\rm out}(\sigma)^2$ で全 σ の実効損失を均一化。学習ノイズ分布 $p_{\rm train}(\sigma)$ を**対数正規（$P_{\rm mean}=-1.2,P_{\rm std}=1.2$）**にして中間ノイズレベルに学習を集中（[[noise-schedule]]）。non-leaky augmentation（augment パラメータを条件入力にし推論時 0）で過学習を防ぐ。config A→F で FID を段階改善（表2）。

## 実験結果と知見

- **CIFAR-10**：条件付き FID **1.79**・無条件 **1.97**（従来 1.85/2.10 を更新）、35 NFE・決定論的・小ネット。
- **ImageNet-64**：サンプラー改善のみで 2.07→**1.55**、学習改善で再学習して **1.36**（従来 SOTA 1.48 を更新）。
- **サンプリングは学習と直交**：事前学習モデルにサンプラーを drop-in 置換するだけで大幅改善。
- **確率性はモデルが良くなると不要に**：CIFAR-10 の改善モデルでは決定論的が最良、確率サンプリングは有害。一方 ImageNet-64（多様）では確率サンプリングが依然有益——データセットの多様性に依存。
- **指標と人手**：FID で評価。$\rho=7$ が生成品質（FID）で広く良好。

## 限界・批判的視点

- **ハイパラのケース依存**：確率サンプリングの $\{S_{\rm churn},S_{\rm tmin},S_{\rm tmax},S_{\rm noise}\}$ はグリッドサーチが必要で、確率サンプリングをモデル評価の主手段にすると学習・アーキ設計に意図せず影響しうる、と著者自身が懸念。
- **高解像度への一般化**：多くのパラメータ値は高解像度データセットでは再調整が要るかもしれない（本論文は 32²/64² 中心）。
- **非保存的ベクトル場**：実践的 denoiser が誘導する場は厳密には保存的でなく、過剰な churn が劣化を招く根本理由は未解明（付録 E）。
- **確率性と学習目的の相互作用**は今後の課題。

## 用語と略称

- **EDM** = Elucidating the Design space of diffusion Models（Karras ら 2022）。慣例的に EDM はこの論文の設計・モデルを指す。
- **denoiser $D(\boldsymbol{x};\sigma)$** = ノイズレベル σ のノイズ除去関数。スコアと $\nabla\log p=(D-\boldsymbol{x})/\sigma^2$ で関係。
- **preconditioning** = ネット入出力の σ 依存スケーリング $c_{\rm skip}/c_{\rm out}/c_{\rm in}/c_{\rm noise}$。**$\sigma_{\rm data}$** = データ標準偏差（CIFAR で 0.5）。
- **Heun 法** = 2 次 Runge-Kutta（台形則）。**ρ スケジュール** = 時刻離散化の多項式 warp（ρ=7）。**churn / $S_{\rm churn}$ 等** = 確率サンプリングのノイズ攪拌制御。
- **VP / VE** = Variance Preserving（DDPM 系）/ Variance Exploding（NCSN 系）。**iDDPM** = improved DDPM（cosine スケジュール）。**DDIM** = 決定論サンプラー。
- **NFE** = Number of Function Evaluations（ネット評価回数＝サンプリングコスト）。**FID** = Fréchet Inception Distance。**$P_{\rm mean}/P_{\rm std}$** = 学習ノイズ対数正規分布のパラメータ。**EMA** = 指数移動平均。

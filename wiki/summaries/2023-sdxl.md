# SDXL: 高解像度画像合成のための潜在拡散モデルの改良

> 原典: [[translations/2023-sdxl]] ・ `raw/papers/SDXL_ ...md`
> 著者・年・会議: Podell, English, Rombach ら（Stability AI）・2023・arXiv:2307.01952（後に主要オープン T2I モデルとして広く利用）

## 一言まとめ

**Stable Diffusion を「3 倍大きい UNet ＋ 2 つのテキストエンコーダ ＋ 新しい条件付け（size/crop/aspect-ratio）＋ refinement model」で大幅強化したオープンな text-to-image 潜在拡散モデル**。アーキテクチャ・データ前処理・2 段サンプリングの実務的改良を積み上げ、人間評価でブラックボックス SOTA（Midjourney 等）に匹敵する品質を、コード・重み公開のまま達成した。

## 背景と問題意識

[[latent-diffusion]]（LDM, Stable Diffusion の基盤。画像を VAE で潜在空間に圧縮してから拡散する）は強力だが、初期の Stable Diffusion（1.x/2.x）には実務上の弱点があった。SDXL はそれらを正面から潰す「エンジニアリング論文」である。狙いは 2 つ：(1) **オープンモデルでブラックボックス（DALL·E 2, Midjourney 等）に追いつく**——透明性が再現性・公平なバイアス評価に不可欠という主張、(2) **個別の品質バグ（被写体の頭切れ、低解像度データの廃棄、正方形しか出ない）を条件付けの工夫で解消**する。新しい理論ではなく、既存 LDM への**モジュラーな改良の積み上げ**が貢献。

前提知識として、LDM は「最小画像サイズ」を要する（VAE の 2 段構造ゆえ）。従来は小さい画像を捨てるかアップスケールしていたが、前者は学習データの大幅減（SDXL の事前学習データでは 256² 未満を捨てると 39% 喪失）、後者はぼけアーティファクトを招く。SDXL の micro-conditioning はこのジレンマを「捨てずに条件として渡す」ことで回避する。

## 提案手法 / 主張

### 1. アーキテクチャとスケール（§2.1）

- **UNet を 860M → 2.6B（約 3×）に大型化**。増加分は主に attention block の増加と、2 つ目のテキストエンコーダによる大きな cross-attention コンテキスト。
- **transformer block の不均一分布 `[0, 2, 10]`**：最高解像度レベルの transformer block を省略し、低レベルに計算を集中（simple diffusion の知見）。最低レベル（8× ダウンサンプリング）は撤去。channel mult. も `[1,2,4]` に短縮。これは [[diffusion-model-architecture]] の「U-Net をどう配分するか」の系譜にあたる改良。
- **2 つのテキストエンコーダ**：CLIP ViT-L ＋ OpenCLIP ViT-bigG の最後から 2 番目の出力を channel 方向に連結（context dim 2048）。さらに OpenCLIP の **pooled text embedding**（文全体を 1 ベクトルに要約したもの）を、timestep embedding に足す追加条件にする（GLIDE 流）。

### 2. Micro-Conditioning（§2.2）— 学習メタデータを条件にする

「いかなる追加教師も要らない」2 つの条件付け。元画像のメタデータを **Fourier 特徴で埋め込み、timestep embedding に加算**する（[[controllable-generation]] のスコア勾配ガイダンスとは別系統の、学習時メタデータによる条件付け）。

- **size conditioning** $c_{size}=(h_{orig}, w_{orig})$：元解像度を条件化。小画像も捨てずに学習でき、推論時は「見かけ解像度」を制御できる（大きい size で条件付けると高品質に。図3）。ImageNet 512² で FID 43.84→36.53（表2）。
- **crop conditioning** $c_{crop}=(c_{top}, c_{left})$：学習中の random crop 座標を条件化。従来 SD は random crop が生成物に漏れ「頭切れ」を起こしたが、推論時に (0,0) を与えると **object-centered**（中央に収まった）画像になる（図4・5）。conditioning-augmentation の一種。

### 3. Multi-Aspect Training（§2.3）

データをアスペクト比 **bucket**（ピクセル数 ≈1024²、辺は 64 の倍数）に分け、バッチを同一 bucket で構成、target size $c_{ar}$ も条件化。正方形以外（16:9 など）を自然に生成。pretrain 後の finetune 段階として適用。

### 4. 改良 VAE ＋ 2 段パイプライン（§2.4–2.5）

- **改良オートエンコーダ**：同じ VAE アーキを大バッチ（256 vs 9）＋EMA でゼロから再学習し、再構成指標（PSNR/SSIM/LPIPS/rFID）を全面改善（表3）。局所・高周波の細部を底上げ。
- **base ＋ refinement model の 2 段**：base SDXL（256²→512²→マルチアスペクト ~1024² で段階学習、offset-noise 0.05）が出した潜在に、同じ潜在空間の別 LDM（先頭 200 ノイズスケールに特化）で **[[image-inpainting]] でなく SDEdit 流の noising-denoising**（一度ノイズを足して same prompt で再ノイズ除去）をかけ、背景や顔の局所品質を上げる。オプションだが効果大。

## 実験結果と知見

- **ユーザー選好（図1）**：4 モデルから選好を取ると SDXL+refiner 48.4%・SDXL base 36.9%・SD1.5 7.9%・SD2.1 6.7%。SDXL が旧 SD を圧倒し、refiner がさらに上積み。
- **vs Midjourney v5.1（図9–11, 付録E）**：PartiPrompts で 17,153 比較、プロンプト追従で SDXL が 54.9% と僅差で上回る。6 カテゴリ中 4、10 チャレンジ中 7 で同等以上。
- **FID は逆に悪化（付録F・図12）**：SDXL の COCO zero-shot FID は SD1.5/2.1 より**悪い**のに人間は SDXL を明確に好む。著者は「**COCO zero-shot FID は美的品質と負の相関**、基盤 T2I の評価は人手で行うべき」と主張（Pick-a-pic の知見を裏付け）。指標と人間評価の乖離を示す重要な論点。

## 限界・批判的視点

- **手の生成**（解剖学の高分散）、**concept bleeding（概念のにじみ・属性混線）**——「青い帽子＋赤い手袋」が「青い手袋＋赤い帽子」になる等。原因は単一トークンに全情報を圧縮する事前学習テキストエンコーダにあると示唆（図7・8、付録B）。
- **長文テキスト描画**は依然不安定（ランダム文字）。
- **2 段で重い**：base と refiner の 2 モデルをメモリに載せる必要があり、アクセシビリティとサンプリング速度を損なう（§3 で単段化を今後の課題に）。
- **transformer 化は当時利得なし**：UViT/DiT を試したが即座の利点は見出せず（[[diffusion-model-architecture]] の DiT とは対照的に、SDXL は改良 U-Net に留まった）。離散時間学習で offset-noise を要し、EDM の連続時間定式化を将来候補に挙げる。

## 用語と略称

- **SDXL** = Stable Diffusion XL。**LDM** = Latent Diffusion Model（[[latent-diffusion]]）。**VAE** = Variational Autoencoder。
- **micro-conditioning** = 学習メタデータ（元サイズ・crop 座標・アスペクト比）を Fourier 埋め込みで条件化する手法。**Fourier feature embedding** = スカラを正弦波で多次元に埋め込む（timestep embedding と同形式）。
- **pooled text embedding** = テキストエンコーダの文全体要約ベクトル。**OpenCLIP ViT-bigG / CLIP ViT-L** = SDXL が併用する 2 つのテキストエンコーダ。
- **multi-aspect training** = アスペクト比 bucket での finetune。**refinement model** = base の潜在を SDEdit で精緻化する第 2 段 LDM。**SDEdit** = ノイズを足して条件付き再ノイズ除去する img2img/編集手法。**offset-noise** = 低周波の色バランス改善のためのノイズ補正。
- **EDM** = Elucidating the Design Space of Diffusion Models（Karras ら 2022, 連続時間定式化）。**DSM** = Denoising Score Matching。**CFG** = Classifier-Free Guidance（[[classifier-free-guidance]]）。
- **FID** = Fréchet Inception Distance、**IS** = Inception Score、**rFID** = 再構成 FID、**SSIM/LPIPS/PSNR** = 再構成品質指標、**CLIP-score** = テキスト整合指標。**concept bleeding** = 複数概念の属性が混線する失敗。

## 既存知識との接続

- [[latent-diffusion]]：SDXL は LDM/Stable Diffusion の直接の後継・大型化版。VAE 潜在空間での拡散という枠組みはそのまま、規模・条件付け・前処理・2 段サンプリングを実務的に改良した。本 wiki では専用ページを設けず latent-diffusion の代表的後継モデルとして扱う。
- [[diffusion-model-architecture]]：UNet の 3× 大型化と transformer block の不均一配分（`[0,2,10]`）は「改良 U-Net」系譜の延長。SDXL は DiT 的な全 transformer 化を試して当時利得を得られず、改良 U-Net に留まった点で、ADM→SDXL→DiT の対比が際立つ。
- [[controllable-generation]]：micro-conditioning は「学習時メタデータを条件化する」タイプの可制御生成で、ControlNet（空間条件）やスコア勾配ガイダンスとは別系統。crop 条件で生成物のフレーミングを制御できる。
- [[text-to-image-generation]]：2 テキストエンコーダ＋pooled embedding による条件付け強化は高解像 T2I の実装上の到達点。
- [[classifier-free-guidance]]：SDXL も CFG でサンプル品質↔多様性を制御（付録C にスコア視点の定式化）。
- [[subject-driven-generation]] / [[low-rank-adaptation]] / [[lora-merging]]：SDXL は DreamBooth / LoRA / ZipLoRA の事実上の標準 base モデル。特に ZipLoRA（[[summaries/2024-ziplora]]）は「SDXL が単一画像でスタイルを学習できる」性質に全面依存する。

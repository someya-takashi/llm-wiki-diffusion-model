# Scalable Diffusion Models with Transformers（DiT）

> 原典: [[translations/2023-dit]] ・ `raw/papers/Scalable Diffusion Models with Transformers.md`
> 著者・年・会議: William Peebles, Saining Xie（UC Berkeley）・2023・ICCV 2023（arXiv 2212.09748）

## 一言まとめ

拡散モデルの「事実上の標準」バックボーンだった畳み込み **U-Net を、潜在パッチ上で動く Transformer（ViT ベース）に置き換えた** 論文。これを **DiT（Diffusion Transformer）** と呼ぶ。U-Net の帰納バイアスは性能に必須でないこと、そして **計算量（Gflops）を増やすほど FID が下がる**という明確なスケーリング則を示し、最大の **DiT-XL/2** が class-conditional ImageNet 256 で **FID 2.27**（当時 SOTA）を達成した。

## 背景と問題意識

DDPM（[[denoising-diffusion]]）以来、拡散モデルは**例外なく畳み込み U-Net** をノイズ予測ネットワークに使ってきた（[[diffusion-model-architecture]]）。一方、言語・視覚・強化学習など他のほぼ全領域は **Transformer** に統一され、「モデル・データ・計算を増やすほど性能が上がる」スケーリング則の恩恵を受けてきた。拡散モデルだけがこの潮流から取り残されていた。

著者の問いは単純：「**拡散モデルに U-Net は本当に必要か？**」。もし Transformer に置き換えられれば、他領域のベストプラクティス・学習レシピ・スケーラビリティをそのまま輸入でき、領域横断の研究も開ける。前提として、拡散の定式化（ε 予測＋学習分散 Σ、ADM 流）、**CFG**（[[classifier-free-guidance]]）、**LDM**（VAE 潜在空間での拡散, [[latent-diffusion]]）はすべて既存のものをそのまま使い、**変えるのはバックボーンだけ**にする。

## 提案手法 / 主張

Stable Diffusion の VAE 潜在空間（256×256 画像 → 32×32×4 潜在）で動く Transformer を設計する（図3）。

### patchify（§3.2）

潜在を patch size $p$（=2/4/8）でパッチに切り、線形埋め込み＋ViT の sine-cos 位置埋め込みでトークン列にする。トークン数 $T=(I/p)^2$。**$p$ を半分にすると $T$ が 4 倍＝Gflops が 4 倍**になる一方、パラメータ数はほぼ不変——この性質が後の「Gflops こそが鍵」という発見の土台。最後は線形デコーダで各トークンを $p{\times}p{\times}2C$（ノイズ＋対角共分散）に復号。

### DiT block と adaLN-Zero（§3.2, §5）

時刻 $t$・クラス $c$ の条件をどう注入するかで 4 種のブロックを比較：

- **in-context**：$t,c$ を追加トークンとして列に足す（最小コスト）。
- **cross-attention**：$t,c$ を別系列にして cross-attention で注入（+15% Gflops と最も重い）。
- **adaLN**：LayerNorm のスケール $\gamma$・シフト $\beta$ を $t,c$ から回帰（adaptive layer norm）。
- **adaLN-Zero**：adaLN に加え、残差接続の直前に次元ごとスケール $\alpha$ を回帰し、**$\alpha$=0 初期化で各ブロックを恒等写像から始める**（ResNet/拡散 U-Net のゼロ初期化に倣う）。

結果、**adaLN-Zero が最良**（cross-attention・in-context を上回り、しかも最軽量）。「条件付け機構と初期化がモデル品質を決定的に左右する」ことを示した。以降の全モデルが adaLN-Zero を使う。

### スケーリング設計空間（§4, §5）

ViT 流に S/B/L/XL（0.3〜118.6 Gflops）× patch size 2/4/8 の 12 モデルを学習。学習は ADM のハイパラをほぼ流用（warmup も正則化も不要、損失スパイクなしで安定）。

## 実験結果と知見

- **Gflops こそが鍵**（図2・図8）：モデルサイズ↑ でも patch size↓ でも、**Gflops を増やすと FID が一貫して下がる**（強い負の相関）。総 Gflops が同じなら異なる構成でも FID が近い（例 DiT-S/2 ≈ DiT-B/4）。パラメータ数は品質を一意に決めない。
- **大モデルほど compute 効率的**（図9）：小モデルを長く学習するより、大モデルを短く学習する方が効率的。
- **adaLN-Zero ≫ cross-attn ≈ in-context**（図5）。
- **SOTA**（表2・表3）：DiT-XL/2-G（cfg=1.5）が **ImageNet 256 で FID 2.27**（LDM の 3.60 を更新、StyleGAN-XL の 2.30 も僅かに上回る）、**512 で 3.04**（ADM の 3.85 を更新）。しかも 118.6 Gflops で、ADM（1120 Gflops）やピクセル空間 U-Net より遥かに compute 効率が良い。recall も LDM より高い。
- **サンプリング計算はモデル計算を代替できない**（図10）：小モデルがサンプリングステップを増やしても、大モデルとの差は埋まらない。
- **付録**：部分チャネル CFG（4 チャネルのうち 3 チャネルだけにガイダンスを当てても、スケール調整で同等の FID）。スケーリングは FID 以外（IS・Precision 等）でも成立し、訓練損失も言語モデルと同様に低く飽和する。VAE デコーダ（original/ft-MSE/ft-EMA）の差は小さい。

## 限界・批判的視点

- **意義 > 限界**：論文自体は限界節を設けず、むしろ「U-Net 不要・Transformer でスケールする」という前向きな主張が核。
- DiT は本論文では class-conditional ImageNet（ラベル条件）に限定。テキスト条件付き生成は将来課題とされた。
- **後世での重要性**：DiT は結論で「DALL·E 2 / Stable Diffusion の drop-in バックボーンになりうる」と予言し、実際に **Stable Diffusion 3・PixArt-α・Sora（動画）** など後続の主要モデルが DiT 系アーキテクチャを採用した。拡散モデルのバックボーンを U-Net から Transformer へ転換させた転換点（本 wiki 未取り込みの後続）。
- 計算量を主軸にした分析ゆえ、small モデルや低リソース環境での最適設計には別の指標が要る（黄金の複雑さ指標は未確定と著者も認める）。

## 用語と略称

- **DiT** = Diffusion Transformer（U-Net の代わりに Transformer をバックボーンに使う拡散モデル）。命名は構成＋パッチサイズで「DiT-XL/2」（XLarge, p=2）。
- **ViT** = Vision Transformer（画像をパッチ列として扱う Transformer）。**patchify** = 画像／潜在をパッチに切ってトークン列にする操作。
- **adaLN** = adaptive Layer Norm（条件から LayerNorm のスケール・シフトを回帰）。**adaLN-Zero** = 各ブロックを恒等写像で初期化する adaLN の変種（最良）。
- **Gflops** = forward pass の浮動小数点演算量（10⁹ 回）。本論文のモデル複雑さの主指標。
- **CFG** = Classifier-Free Guidance（[[classifier-free-guidance]]）。**LDM** = Latent Diffusion Model（[[latent-diffusion]]）。**VAE** = Variational Autoencoder（潜在のエンコーダ・デコーダ）。
- **FID** = Fréchet Inception Distance、**sFID** = 空間特徴版、**IS** = Inception Score、**EMA** = 重みの指数移動平均。

## 既存知識との接続

- [[diffusion-model-architecture]]：**DiT は本概念の第 2 の柱**。「改良 U-Net（ADM）」に対し「Transformer 化（DiT）」というバックボーン設計の転換を示した。
- [[latent-diffusion]]：DiT は LDM の枠組み（Stable Diffusion の VAE 潜在空間）でバックボーンだけを Transformer 化したもの。VAE は既製を流用。
- [[denoising-diffusion]]：拡散の数学（ε 予測・$L_\text{simple}$・学習分散 Σ）と ADM のハイパラをそのまま継承し、ネットワークだけ差し替えた。
- [[classifier-free-guidance]]：DiT も CFG で高品質化（cfg=1.5 で SOTA）。部分チャネル CFG の知見も提示。

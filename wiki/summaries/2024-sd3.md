# 高解像度画像合成のための Rectified Flow Transformer のスケーリング（Stable Diffusion 3）

> 原典: [[translations/2024-sd3]] ・ `raw/papers/Scaling Rectified Flow Transformers ...md`
> 著者・年・会議: Esser, Kulal, Rombach ら（Stability AI）・2024・ICML 2024（arXiv:2403.03206。通称 **Stable Diffusion 3 / SD3**）

## 一言まとめ

**Stable Diffusion 3（SD3）の技術レポート**。(1) **rectified flow（整流フロー）**——データとノイズを直線で結ぶ [[flow-matching]] 系の定式化——を、中間時刻に重みを置く新しいノイズサンプラー（logit-normal・mode）で改良し、大規模比較で従来の拡散定式化（EDM・LDM-Linear）を上回ることを示す。(2) テキストと画像に**別々の重み**を使い attention でのみ結合する新アーキテクチャ **MM-DiT** を提案。(3) 8B パラメータまでのスケーリング研究で「**検証損失が人間評価・ベンチマークと強く相関**」することを示し、SDXL・DALL-E 3 を上回る。

## 背景と問題意識

拡散モデルは「データ→ノイズ」の順過程を反転して生成する。順過程の経路の選び方がサンプリング効率を左右する：**曲がった経路**（従来の拡散）は多くの積分ステップ（＝ニューラルネット評価）を要するが、**直線経路**なら少ステップ・誤差蓄積も小さい。この直線経路を使うのが **Rectified Flow（RF）** で、$z_t=(1-t)x_0+t\epsilon$ という単純な線形補間でデータ $x_0$ とノイズ $\epsilon$ を結び、速度場 $v_\Theta$ を回帰する（[[flow-matching]] / [[probability-flow-ode]] の系譜）。理論的に綺麗だが、それまで大規模 text-to-image では決定的に確立されていなかった。

もう 1 つの問題意識は**条件付けアーキテクチャ**。従来の text-to-image（[[latent-diffusion]] 系）は固定テキスト表現を cross-attention で一方向に注入するだけだった。SD3 は、テキストと画像が双方向に情報をやり取りすべきだと主張する。

## 提案手法 / 主張

### 1. Rectified Flow ＋ 調整された時刻サンプラー

RF 損失は全時刻を一様に学習するが、速度予測目標 $\epsilon-x_0$ は**中間時刻ほど難しい**（端点では最適予測が単に平均）。そこで時刻分布を一様 $\mathcal{U}(t)$ から、中間に重みを置く密度 $\pi(t)$ に変える（これは重み付き損失 $w_t^\pi=\frac{t}{1-t}\pi(t)$ と等価）。

- **logit-normal サンプリング** $\pi_{\text{ln}}(t;m,s)$：logit 変換した時刻を正規分布でサンプル。位置 $m$ でデータ／ノイズ寄りに偏らせ、スケール $s$ で幅を制御。
- **mode サンプリング**：端点でも密度が消えない代替。
- **CosMap**：cosine スケジュールの log-SNR を RF に移植。

61 定式化の大規模比較（ImageNet・CC12M、CLIP/FID/検証損失、非優越ソート）で **rf/lognorm(0.00, 1.00) が一貫して最良**。一様 RF（rf）や LDM-Linear（eps/linear）を上回り、特に少ステップ（5 ステップ）でも品質低下が小さい。

### 2. MM-DiT（Multimodal Diffusion Transformer）

[[diffusion-model-architecture]] の DiT を土台に、**テキストと画像で別々の重みの組**を持つ transformer ブロック。各モダリティは自分の重みで処理されるが、**attention 演算のときだけ両系列を結合**し、双方向に情報を混ぜる（実質「2 つの独立 transformer が attention で握手する」）。時刻 $t$ と pooled テキスト $c_{\text{vec}}$ は adaLN 風の変調に、系列テキスト $c_{\text{ctxt}}$ はトークン列として入る。テキストは **3 エンコーダ**（CLIP L/14・OpenCLIP bigG/14・T5-XXL）で符号化。

実験（図4）で MM-DiT は vanilla DiT・CrossDiT（cross-attention 型）・UViT を検証損失・CLIP・FID で明確に上回る。DiT は「全モダリティ 1 重み共有の MM-DiT 特殊ケース」と位置づけられる。

### 3. スケールでの学習と諸改良

- **QK-normalization**：高解像度で mixed-precision 学習が発散する問題を、attention 前に Q・K を RMSNorm することで防ぐ（attention logit 増大不安定性の抑制）。
- **解像度依存の時刻シフト**：高解像度ほど信号破壊に多くのノイズが要るので、低解像度の時刻 $t_n$ を高解像度の $t_m$ に写像（log-SNR を $\log\frac{n}{m}$ シフト）。$1024^2$ では shift $\alpha=3.0$。
- **改良オートエンコーダ**（潜在 16 チャネル）、**合成キャプション**（CogVLM で 50/50 混合、GenEval 改善）、**DPO**（LoRA でランク 128 の選好最適化）。
- **スケーリング則**：8B・$5\times10^{22}$ FLOPs まで、検証損失がモデルサイズ・ステップで滑らかに減少し**飽和しない**。検証損失が GenEval・T2I-CompBench・人間選好と強く相関——「検証損失が性能の単純な予測子」。

## 実験結果と知見

- **GenEval（表5）**：最大モデル depth=38（1024²・DPO）が総合 0.74 で、DALL-E 3（0.67）・SDXL（0.55）を上回る。
- **人間選好（図7）**：8B モデルが視覚的美しさ・プロンプト追従・タイポグラフィで現行 SOTA（プロプライエタリ含む）を上回るか同等。
- **大モデルは少ステップで足りる（表6）**：depth が大きいほど経路長が短く（より直線的）、5/50 ステップでの相対 CLIP 低下が小さい。
- **T5 の役割**：複雑・長文プロンプトでは T5 が効くが、多くのプロンプトでは推論時に T5 を外しても競争力を維持（メモリ節約に有用）。

## 限界・批判的視点

- **RF は最適輸送解ではない**：直線補間は学習が単純だが厳密な最適輸送ではなく、軌道曲率をさらに減らす後続研究の余地がある（付録A 自認）。
- **2 段でなく単段だが巨大**：8B＋T5-XXL（4.7B）で VRAM 要求が大きい。T5 ドロップで緩和。
- **データフィルタ依存**：性能は合成キャプション・重複除去（SSCD）・美的フィルタといったデータ工学に強く依存し、再現は容易でない。
- **記憶（memorization）リスク**：SSCD 重複除去で記憶サンプルを 5× 削減するが、生成モデル一般の課題は残る。

## 用語と略称

- **SD3** = Stable Diffusion 3。**RF** = Rectified Flow（整流フロー、$z_t=(1-t)x_0+t\epsilon$ の直線経路）。
- **MM-DiT** = Multimodal Diffusion Transformer（テキスト／画像に別重み、attention で結合）。**DiT** = Diffusion Transformer（[[diffusion-model-architecture]]）。**UViT** = U-Net と transformer のハイブリッド。
- **Flow Matching / Conditional Flow Matching (CFM)** = ベクトル場 $u_t$ をシミュレーション不要で回帰する目的（[[flow-matching]]）。**SNR** = Signal-to-Noise Ratio（$\lambda_t=\log(a_t^2/b_t^2)$）。
- **logit-normal / mode / CosMap サンプラー** = 学習時刻 $t$ の分布。**QK-normalization** = attention 前の Q・K の RMSNorm。
- **EDM** = Karras らの連続時間拡散定式化。**EMA** = 指数移動平均。**DPO** = Direct Preference Optimization（選好最適化）。**CFG** = Classifier-Free Guidance（[[classifier-free-guidance]]）。
- **CogVLM** = 合成キャプション用 VLM。**T5-XXL / CLIP / OpenCLIP** = テキストエンコーダ。**SSCD** = 重複検出モデル。**GenEval / T2I-CompBench / FID / CLIP-score** = 評価指標。

## 既存知識との接続

- [[flow-matching]]：SD3 は rectified flow（flow matching の直線経路インスタンス）を**大規模 text-to-image で初めて決定的に確立**した。中間時刻に重みを置くサンプラーが核心の貢献で、flow matching の実用化におけるランドマーク。
- [[diffusion-model-architecture]]：MM-DiT は DiT の自然な発展形（マルチモーダル化）。DiT＝1 重み共有の特殊ケース。QK-normalization と並んで、本 wiki の「U-Net→DiT→MM-DiT」アーキテクチャ系譜の最新地点。
- [[latent-diffusion]]：SD3 は LDM/SDXL（[[summaries/2023-sdxl]]）の後継。VAE 潜在空間（16 チャネル）で動作し、SDXL の micro-conditioning 思想を引き継ぎつつ、バックボーンを MM-DiT に、定式化を RF に置き換えた。
- [[text-to-image-generation]]：3 テキストエンコーダ（CLIP×2＋T5）と双方向 attention でテキスト理解・タイポグラフィを強化。
- [[probability-flow-ode]]：RF の生成 ODE $dy_t=v_\Theta dt$ は確率フロー ODE の一種で、直線経路ゆえ少ステップ Euler 法で解ける。
- [[low-rank-adaptation]]：DPO による選好最適化を LoRA（ランク 128）で実装。
